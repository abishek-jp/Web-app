How to Execute the Project

1. Click the download button on the main page to download our Jupyter Notebook.
2. Open the notebook using Jupyter Notebook or Jupyter Lab.
3. Run each cell sequentially (data processing → encoding → training → prediction).
4. The XGBoost model will be trained and ready to generate crop yield predictions.

Correct Flow to Execute

  1. Download our notebook from the main page.
  2. Load the dataset inside the notebook.
  3. Apply OneHotEncoding to the categorical features (State, Season, Crop).
  4. Split the dataset into training and testing sets.
  5. Train the XGBoost model using the processed dataset.
  6. Provide new input values for prediction.
  7. Interpret the predicted crop production output.

How to Give Input

Inside our prediction cell, provide the following fields:
State (e.g., Karnataka)
Season (Kharif / Rabi / Whole Year)
Crop (Rice, Wheat, Maize, etc.)
Area (in hectares)

Our notebook applies OneHotEncoding to the categorical fields and then feeds the processed values to the model.

What is Processed

• Our input fields are encoded using OneHotEncoder.
• All encoded and numerical features form a combined feature vector.
• This feature vector is passed into our XGBoostRegressor model.
• The model uses boosted decision trees to calculate the predicted crop production.

How to Interpret the Results

Our model outputs the predicted production value (in tons or kilograms depending on the dataset).
A larger numerical value indicates a higher expected yield for the given inputs.
By adjusting values such as State, Season, Crop, or Area, we can observe their impact on predicted production and compare different agricultural scenarios.
